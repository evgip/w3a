# 🌐 W3A — Новостной агрегатор в стиле Hacker News и Lobsters

🇷🇺 Русский | 🇬🇧 [English](README.md)

Современный, легковесный новостной агрегатор, созданный на **PHP 8.1+** и **MySQL**, вдохновлённый [Hacker News](https://news.ycombinator.com) и [Lobste.rs](https://lobste.rs). Модульная HMVC-архитектура, собственный DI-контейнер, событийная система, RSS-фиды, система закладок и игнорирования пользователей, комплексные инструменты модерации.

## 🚀 Возможности

### Основной функционал
- **Новости** — Публикация новостей с URL или текстовым содержимым
- **Древовидные комментарии** — Вложенные комментарии без ограничения глубины
- **Глобальная лента комментариев** — Страница `/comments` со всеми свежими комментариями сайта (как в Lobsters)
- **Комментарии пользователя** — Страница `/user/{username}/comments` со всеми комментариями пользователя
- **Публикации пользователя** — Страница `/user/{username}/stories` с историей публикаций
- **Система голосований** — Голосование за/против; для минуса требуется минимальная карма
- **Аутентификация** — Регистрация, вход, «Запомнить меня», активация по e-mail, восстановление пароля
- **Профили пользователей** — О себе, аватар, карма, лента активности
- **Личные сообщения** — Приватные диалоги между пользователями
- **Уведомления** — Внутренние уведомления об ответах, комментариях в отслеживаемых историях, упоминаниях, действиях модераторов
- **Wiki для тегов** — Вики-страницы, привязанные к тегам, с историей правок и правами редактирования
- **Предложения правок** — Пользователи могут предлагать изменения к историям (теги, заголовок), которые проходят модерацию

### Персонализация и отслеживание
- **Ленты прочтения (Read Ribbons)** — Отслеживание прочитанных комментариев в каждой истории; бейджи "+N новых" в ленте
- **Подсветка новых комментариев** — Визуальное выделение непрочитанных комментариев в историях
- **Глобальное отслеживание комментариев** — Разделитель "↑ Новые комментарии ↓" на странице `/comments`
- **Закладки (Saved Stories)** — Сохранение историй в личный список "почитать позже" на странице `/saved`
- **Игнорирование пользователей (Muting)** — Скрытие историй, комментариев и уведомлений от замьюченных пользователей
- **Фильтры тегов** — Пользователи могут скрывать истории по неинтересным тегам
- **Подписка на истории** — Уведомления о новых комментариях в отслеживаемых историях
- **Настройки уведомлений** — Гибкая настройка: e-mail, внутренние уведомления, по типам событий

### RSS-фиды
- `/rss` — Все новые истории
- `/t/{tag}/rss` — Истории по конкретному тегу
- `/u/{username}/rss` — Истории пользователя
- `/comments/rss` — Все новые комментарии
- Автообнаружение RSS в браузерах через `<link rel="alternate">`

### Ранжирование и поиск
- **Алгоритм горячести** — Собственная формула, учитывающая очки, возраст и весовой модификатор тега
- **Интервал Уилсона** — Статистически обоснованное ранжирование комментариев по достоверности
- **Система тегов** — Истории тегируются; теги объединяются в категории
- **Полнотекстовый поиск** — Поиск по новостям и комментариям

### Модерация и доверие
- **Система приглашений** — Можно отключить открытую регистрацию; приглашать могут только пользователи с нужной кармой
- **Система жалоб (флагов)** — Пользователи могут жаловаться на контент; при достижении порога контент скрывается автоматически
- **Управление доменами** — Бан целых доменов; истории с забаненных доменов отклоняются автоматически
- **Баны пользователей** — Временные или перманентные баны с указанием причины
- **Баны по IP** — Блокировка злоупотребляющих IP-адресов
- **Заметки модераторов** — Приватные заметки к аккаунтам пользователей, видны только персоналу
- **Журнал активности модераторов** — Ежедневная статистика модераторских действий
- **Журнал аудита** — Полный след каждого административного действия
- **Журнал модерации (публичный)** — Прозрачность действий модераторов для всех пользователей

### Архитектура и удобство разработки
- **DI-контейнер** — Лёгкий контейнер внедрения зависимостей
- **Диспетчер событий** — Паттерн Observer: `StoryCreated`, `CommentCreated`, `CommentUpdated`, `UserBanned`, `FlagResolved` и др.
- **Конвейер middleware** — HTTP-мидлвари для авторизации, CSRF, rate-limiting, заголовков безопасности
- **Модульный HMVC** — Каждая фича — самодостаточный модуль (Controllers / Models / Views / Services / routes)
- **Сервис-провайдеры** — Регистрация сервисов по модулям
- **Разделение ответственности** — Комментарии, закладки, игнорирование, RSS, wiki — в отдельных модулях

### Безопасность
- **Хеширование паролей** — bcrypt с автоматической солью
- **CSRF-защита** — Проверка токенов на каждой форме, меняющей состояние
- **Защита от SQL-инъекций** — Подготовленные выражения PDO везде
- **Защита от XSS** — Экранирование вывода через `htmlspecialchars()`
- **Rate Limiting** — Настраиваемые лимиты по действиям и IP
- **Безопасность сессий** — HTTP-only cookies, регенерация сессии
- **Валидация ввода** — Серверная проверка на каждом эндпоинте
- **Заголовки безопасности** — Strict-Transport-Security, X-Frame-Options и др.
- **CAPTCHA** — Встроенная CAPTCHA для чувствительных действий
- **Файрвол** — Фильтрация запросов и проверка репутации IP

### Пользовательский опыт
- **Адаптивный дизайн** — Мобильная версия интерфейса
- **Светлая и тёмная темы** — Автоматическое определение по системным настройкам + ручное переключение через `localStorage`
- **CSS-переменные** — Единая система темизации через `:root` и `[data-theme="dark"]`
- **Горячие клавиши** — `C` для сворачивания комментариев и другие
- **Поддержка Markdown** — Форматирование текста в комментариях и новостях (через Parsedown)
- **Сворачивание веток комментариев** — С сохранением состояния в `localStorage`
- **Панель инструментов Markdown** — Кнопки форматирования с предпросмотром

## 📦 Установка

Установка через Composer:

```bash
composer create-project evgip/w3a
```

### Требования
- PHP 8.1 или выше
- MySQL 8.0 или выше
- Apache / Nginx с включённой переадресацией URL
- Composer (опционально, для разработки)

### Шаг 1 — Клонирование репозитория

```bash
git clone https://github.com/evgip/w3a.git
cd w3a
```

### Шаг 2 — Создание базы данных

1. Создайте базу данных MySQL:

```sql
CREATE DATABASE w3a CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. Импортируйте схему:

```bash
mysql -u your_username -p w3a < db/schema.sql
```

### Шаг 3 — Настройка приложения

Переименуйте файл `.env.example` в `.env` и заполните его.

### Шаг 4 — Установка прав доступа

```bash
chmod -R 755 storage/
chmod -R 755 public/
```

### Шаг 5 — Настройка веб-сервера

**Apache** (`.htaccess` уже включён):

```apache
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ index.php [QSA,L]
```

**Nginx**:

```nginx
location / {
    try_files $uri $uri/ /index.php?$query_string;
}
```

### Шаг 6 — Учётные записи по умолчанию

| Роль         | E-mail               | Пароль    |
|--------------|----------------------|-----------|
| Администратор| admin@example.com    | password  |
| Пользователь | test@test.ru         | password  |

> ⚠️ Сразу после первого входа смените эти пароли.

## 📊 Схема базы данных

### Контент
| Таблица           | Описание                                              |
|-------------------|-------------------------------------------------------|
| `stories`         | Новости (URL или текст), с оценкой горячести          |
| `comments`        | Древовидные комментарии с оценкой достоверности       |
| `votes`           | Полиморфные голоса за/против                          |
| `tags`            | Определения тегов с модификатором горячести           |
| `taggings`        | Связь «многие-ко-многим» между историями и тегами     |
| `categories`      | Категории тегов                                       |
| `tag_filters`     | Исключения тегов по пользователям                     |
| `wiki_pages`      | Wiki-страницы, привязанные к тегам                    |
| `wiki_revisions`  | История правок wiki-страниц                           |
| `wiki_permissions`| Права редактирования wiki по тегам                    |
| `suggestions`     | Предложения правок историй (теги, заголовки)          |

### Пользователи и авторизация
| Таблица                | Описание                                         |
|------------------------|--------------------------------------------------|
| `users`                | Учётные записи (роль: user / moderator / admin)  |
| `user_profiles`        | О себе, аватар                                   |
| `user_settings`        | Настройки уведомлений                            |
| `user_bans`            | Временные / перманентные баны                    |
| `banned_ips`           | Блокировки по IP                                 |
| `password_resets`      | Токены восстановления пароля                     |
| `email_activations`    | Токены подтверждения e-mail                      |
| `invitations`          | Коды приглашений                                 |
| `invitation_requests`  | Публичные запросы приглашений                    |

### Социальное и персонализация
| Таблица                | Описание                                         |
|------------------------|--------------------------------------------------|
| `conversations`        | Диалоги личных сообщений                         |
| `messages`             | Отдельные сообщения                              |
| `user_notifications`   | Внутренние уведомления                           |
| `read_ribbons`         | Последний прочитанный комментарий в истории      |
| `saved_stories`        | Закладки пользователей (сохранённые истории)     |
| `muted_users`          | Игнорируемые пользователи                        |

### Модерация
| Таблица          | Описание                                              |
|------------------|-------------------------------------------------------|
| `flags`          | Жалобы на контент (спам, оскорбления и т. д.)         |
| `domains`        | Забаненные / разрешённые домены                        |
| `mod_notes`      | Приватные заметки модераторов о пользователях         |
| `mod_activity`   | Ежедневное количество модераторских действий          |
| `audit_logs`     | Полный журнал административных действий               |

### Инфраструктура
| Таблица          | Описание                                              |
|------------------|-------------------------------------------------------|
| `sessions`       | Активные сессии пользователей                         |
| `rate_limits`    | Счётчики ограничения частоты запросов                 |

Полная схема с индексами и внешними ключами — в файле `db/schema.sql`.
